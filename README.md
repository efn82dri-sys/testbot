# راهنمای کامل ساخت ربات «تایید عضویت»
### رواق | مرجع فایل‌های معماری

این راهنما طوری نوشته شده که حتی اگر تا امروز هیچ کد ننوشته‌اید، بتوانید
قدم‌به‌قدم جلو بروید. فقط ترتیب مراحل را رعایت کنید.

---

## ساختار پروژه

```
telegram-membership-bot/
├── main.py                  ← کد اصلی ربات (بک‌اند)
├── requirements.txt         ← لیست کتابخانه‌های مورد نیاز
├── data/
│   ├── submissions.jsonl    ← اینجا پاسخ فرم‌ها ذخیره می‌شود (خودکار ساخته می‌شود)
│   └── contact_messages.json ← نگاشت پیام‌های فوروارد‌شده به ادمین (خودکار ساخته می‌شود)
└── webapp/
    ├── index.html           ← صفحه‌ی فرم
    ├── style.css            ← ظاهر فرم
    ├── script.js            ← منطق فرم
    └── fonts/                ← اینجا فایل فونت کلمه را قرار می‌دهید
```

---

## مرحله ۱ — ساخت ربات و گرفتن Token از BotFather

1. در تلگرام، کادر جستجو را باز کنید و **BotFather** را پیدا کنید (اکانت آبی‌تیک با یوزرنیم `@BotFather`).
2. دستور `/newbot` را بفرستید.
3. یک اسم نمایشی برای ربات انتخاب کنید (مثلاً «تایید عضویت معماری»).
4. یک یوزرنیم انتخاب کنید که حتماً باید به `bot` ختم شود (مثلاً `ArchMembershipBot`).
5. BotFather یک پیام حاوی **Token** برایتان می‌فرستد؛ چیزی شبیه این:
   ```
   123456789:AAExampleTokenAbCdEfGhIjKlMnOpQrStUv
   ```
   این را جایی امن کپی کنید — بعداً لازمش دارید. **آن را با کسی به اشتراک نگذارید.**

### فعال‌سازی حالت WebApp برای ربات
هنوز داخل چت با BotFather:
1. دستور `/mybots` را بفرستید و ربات‌تان را انتخاب کنید.
2. وارد **Bot Settings → Menu Button** شوید — نیازی نیست چیزی دستی
   تنظیم کنید، چون خودِ ربات هر بار که بالا می‌آید (روی Render)،
   Menu Button را به‌صورت خودکار روی لینک مینی‌اپ تنظیم می‌کند تا
   کاربران بتوانند از کنار جعبه‌ی پیام هم فرم را باز کنند.

---

## مرحله ۲ — ساخت گروه و تنظیم «تایید اعضای جدید»

1. یک گروه تلگرامی بسازید (یا از گروه فعلی‌تان استفاده کنید).
2. ربات ساخته‌شده را به گروه اضافه کنید و آن را **ادمین** کنید. حداقل
   دسترسی‌های لازم برای ربات:
   - `Add Users` (افزودن کاربران)
   - `Invite Users via Link`
3. وارد تنظیمات گروه شوید:
   **Group Settings → Group Type → Approve New Members** را روشن کنید.
   با این کار، هر کسی که بخواهد عضو شود، اول یک «درخواست عضویت»
   می‌فرستد و تا وقتی ربات تاییدش نکند وارد گروه نمی‌شود — دقیقاً
   همان چیزی که در تصاویر نمونه دیدید.
4. آیدی عددی گروه را پیدا کنید (لازم است در تنظیمات ربات وارد کنید):
   - ساده‌ترین راه: پیامی داخل گروه بفرستید و آن را برای ربات
     `@userinfobot` فوروارد کنید؛ آیدی گروه (چیزی شبیه `-1001234567890`)
     را برایتان نشان می‌دهد.
5. یک لینک دعوت عمومی برای گروه بسازید (**Invite Link**) و آن را هم
   نگه دارید؛ در تنظیمات ربات استفاده می‌شود تا بعد از تایید، دکمه‌ی
   «ورود به گروه» به کاربر نشان داده شود.

## مرحله ۳ — نصب فونت «کلمه» (Kalameh)

1. فایل فونت کلمه را با فرمت `woff2` و `woff` تهیه کنید (از یک منبع
   معتبر توزیع فونت فارسی که مجوز استفاده‌ی وب دارد).
2. دو فایل زیر را داخل پوشه‌ی `webapp/fonts/` قرار دهید، دقیقاً با
   همین نام‌ها:
   ```
   webapp/fonts/Kalameh-Regular.woff2
   webapp/fonts/Kalameh-Bold.woff2
   ```
   (فرمت‌های `.woff` هم اگر داشتید کنار همین‌ها بگذارید؛ کد از قبل
   برایشان آماده است.)
3. اگر فونت را نگذارید، مشکلی پیش نمی‌آید — فرم به‌طور خودکار با فونت
   Tahoma نمایش داده می‌شود؛ فقط ظاهر کمی فرق می‌کند.

## مرحله ۴ — تنظیم محیط VS Code (فقط برای تست روی سیستم خودتان)

> توجه: چون Telegram WebApp نیاز به آدرس HTTPS واقعی دارد، اجرای روی
> سیستم شخصی فقط برای بررسی ظاهر صفحه‌ی فرم در مرورگر مفید است، نه
> برای تست کامل داخل تلگرام. برای تست واقعی مستقیم به «مرحله ۵
> (دیپلوی روی Render)» بروید.

1. [Python](https://www.python.org/downloads/) نسخه‌ی ۳.۱۱ یا بالاتر
   را نصب کنید (هنگام نصب تیک **Add Python to PATH** را بزنید).
2. [VS Code](https://code.visualstudio.com/) را نصب کنید و اکستنشن
   **Python** (از مایکروسافت) را از تب Extensions نصب کنید.
3. پوشه‌ی پروژه (`telegram-membership-bot`) را در VS Code باز کنید
   (`File → Open Folder`).
4. یک ترمینال جدید باز کنید (`Terminal → New Terminal`) و این دستورات
   را یکی‌یکی بزنید:
   ```bash
   python -m venv venv
   ```
   سپس برای فعال‌سازی محیط مجازی:
   - ویندوز:
     ```bash
     venv\Scripts\activate
     ```
   - مک / لینوکس:
     ```bash
     source venv/bin/activate
     ```
   و در نهایت نصب کتابخانه‌ها:
   ```bash
   pip install -r requirements.txt
   ```

## مرحله ۵ — دیپلوی روی Render (رایگان و همیشه‌فعال)

از بین سرویس‌های رایگان، **Render** برای این نوع ربات (که همزمان هم
ربات است هم یک صفحه‌ی وب کوچک) از PythonAnywhere ساده‌تر و مناسب‌تر
است، چون امکان اجرای وب‌سرور دائمی با HTTPS رایگان می‌دهد.

1. یک حساب رایگان در [render.com](https://render.com) بسازید.
2. کد پروژه را در یک ریپازیتوری روی GitHub آپلود کنید (اگر با گیت‌هاب
   آشنا نیستید: کافی‌ست یک ریپازیتوری جدید بسازید و فایل‌های پروژه را
   با گزینه‌ی **Upload files** در صفحه‌ی گیت‌هاب آپلود کنید).
3. داخل Render روی **New → Web Service** بزنید و ریپازیتوری‌تان را
   انتخاب کنید.
4. تنظیمات سرویس:
   - **Build Command:**
     ```bash
     pip install -r requirements.txt
     ```
   - **Start Command:**
     ```bash
     python main.py
     ```
5. در بخش **Environment Variables** این مقادیر را اضافه کنید:

   | Key                 | Value                                            |
   |----------------------|--------------------------------------------------|
   | `BOT_TOKEN`          | توکنی که از BotFather گرفتید                     |
   | `GROUP_CHAT_ID`      | آیدی عددی گروه (مثل `-1001234567890`)            |
   | `GROUP_INVITE_LINK`  | لینک دعوت گروه                                   |
   | `WEBHOOK_HOST`       | آدرسی که Render بعد از دیپلوی به شما می‌دهد (مثل `https://arch-bot.onrender.com`) |
   | `NOTIFY_CHAT_ID`     | (اختیاری) آیدی عددی خودتان یا آیدی عددی کانالی که گزارش عضو جدید/ترک عضو و پیام‌های ارتباط‌با‌ادمین برایش ارسال شود |
   | `ADMIN_IDS`          | (اختیاری) آیدی عددی خودتان (و هر ادمین دیگری)، با ویرگول جدا از هم — برای استفاده از `/stats`، `/export`، `/broadcast` و دریافت/پاسخ پیام‌های اعضا |

   نکته درباره‌ی `NOTIFY_CHAT_ID`:
   - اگر آیدی شخصی خودتان را بگذارید، باید قبلش حتماً یک بار به ربات
     `/start` زده باشید، وگرنه تلگرام اجازه‌ی پیام دادن به ربات را
     نمی‌دهد.
   - اگر آیدی یک کانال را بگذارید (چیزی مثل `-1001234567890`)، باید
     ربات را در آن کانال ادمین با دسترسی «ارسال پیام» کنید.
   - اگر این متغیر را خالی بگذارید، گزارش‌های عضو جدید/ترک عضو اصلاً
     ارسال نمی‌شوند (بقیه‌ی قابلیت‌ها مثل تایید عضویت مثل قبل کار
     می‌کنند).

   نکته درباره‌ی `ADMIN_IDS`:
   - آیدی عددی خودتان را همان‌طور که در «مرحله ۲» با `@userinfobot`
     پیدا کردید، اینجا هم می‌توانید استفاده کنید (این بار آیدی خودِ
     اکانتتان را بفرستید، نه گروه را).
   - اگر چند نفر ادمین دارید، آیدی‌ها را با ویرگول جدا کنید، مثل:
     `111111111,222222222`
   - اگر این متغیر را خالی بگذارید، دستورات `/stats`، `/export` و
     `/broadcast` برای هیچ‌کس کار نمی‌کنند.

   نکته: چون آدرس `WEBHOOK_HOST` را فقط بعد از اولین دیپلوی می‌فهمید،
   می‌توانید اول سرویس را با یک مقدار موقت بسازید، آدرس نهایی را از
   بالای صفحه‌ی Render کپی کنید، سپس در Environment Variables اصلاحش
   کنید و روی **Manual Deploy → Deploy latest commit** بزنید تا با
   مقدار درست دوباره بالا بیاید.

6. بعد از چند دقیقه، وضعیت سرویس **Live** می‌شود. یعنی ربات‌تان
   ۲۴ ساعته روشن است.

## مرحله ۶ — تست نهایی

1. با یک اکانت تلگرام دیگر (یا از حساب دوستتان)، وارد لینک دعوت گروه
   شوید و «Request to Join» بزنید.
2. ربات باید ظرف چند ثانیه برای همان اکانت پیام خصوصی بفرستد با دکمه‌ی
   «تکمیل فرم پذیرش».
3. فرم را پر کنید و «ثبت نهایی» را بزنید.
4. باید پیام «✅ عضویت شما تایید شد!» را ببینید و کاربر به‌طور خودکار
   عضو گروه شود.
5. برای دیدن پاسخ‌های ثبت‌شده، وارد پنل Render شوید، از تب **Shell**
   دستور زیر را بزنید تا محتوای فایل را ببینید:
   ```bash
   cat data/submissions.jsonl
   ```

## نکات مهم

- **حریم خصوصی:** فایل `submissions.jsonl` روی سرور Render ذخیره
  می‌شود؛ اگر می‌خواهید داده‌ها را به‌جای فایل، در Google Sheets
  ذخیره کنید، باید یک سرویس Google Cloud با کلید API بسازید — این
  کار به دانش بیشتری نیاز دارد و در صورت تمایل می‌توانم در مرحله‌ی
  بعد آن را هم برایتان اضافه کنم.
- اگر کاربری قبلاً هیچ‌وقت به ربات پیام نداده باشد (`/start` نزده
  باشد)، ممکن است تلگرام اجازه‌ی ارسال پیام خصوصی خودکار را ندهد. در
  این صورت بهتر است در توضیحات گروه از اعضا بخواهید قبل از درخواست
  عضویت، یک بار روی ربات `/start` بزنند.
- برای تغییر متن‌ها، سوالات یا لیست علایق، کافی‌ست فایل‌های
  `webapp/index.html` و `webapp/script.js` را ویرایش کنید — نیازی به
  تغییر `main.py` نیست.
- **قابلیت‌های جدید:** حالا وقتی کسی با موفقیت عضو گروه شود، اگر
  `NOTIFY_CHAT_ID` را تنظیم کرده باشید یک گزارش برایتان ارسال می‌شود.
  همچنین وقتی کسی گروه را ترک کند، ربات برایش (در چت خصوصی) پیام
  می‌دهد، یک نظرسنجی درباره‌ی دلیل ترک گروه می‌فرستد و در پایان با
  یک دکمه او را به بازگشت به گروه دعوت می‌کند. برای اینکه این دو
  قابلیت کار کنند، ربات باید همچنان **ادمین گروه** بماند (نیاز به
  دسترسی خاص اضافه‌ای ندارد).
- **پنل مدیریت شیشه‌ای (فقط برای آیدی‌های داخل `ADMIN_IDS`)** — در
  چت خصوصی با ربات دستور `/admin` را بفرستید تا یک منوی دکمه‌دار
  (Inline) باز شود:
  - **📊 آمار گروه** — تعداد اعضای فعلی، تعداد فرم‌های تکمیل‌شده و
    نرخ ترک گروه را نشان می‌دهد.
  - **📄 خروجی اکسل فرم‌ها** — یک فایل اکسل (`.xlsx`) شامل تمام
    فرم‌های ثبت‌شده برایتان می‌فرستد (اتصال مستقیم به Google Sheets
    نیاز به ساخت یک سرویس Google Cloud با کلید API دارد که دانش
    بیشتری می‌طلبد؛ فعلاً خروجی اکسل جایگزین ساده‌تر آن است — اگر
    بعداً به گوگل‌شیت نیاز داشتید بگویید تا اضافه‌اش کنم).
  - **📢 ارسال پیام همگانی** — متن پیام را از شما می‌پرسد، یک
    پیش‌نمایش همراه با تعداد گیرندگان نشان می‌دهد، و فقط بعد از
    تایید شما (دکمه‌ی «✅ ارسال شود») پیام را برای همه‌ی کسانی که
    تا الان فرم را پر کرده‌اند می‌فرستد و گزارش موفق/ناموفق می‌دهد.
  - **❌ بستن** — پنل را می‌بندد.

  همین سه قابلیت با دستورات متنی هم در دسترس‌اند، اگر ترجیح می‌دهید
  تایپ کنید: `/stats`، `/export`، و `/broadcast متن پیام شما`.

- **ارتباط با ادمین (جدید):** هر عضو می‌تواند در چت خصوصی با ربات
  دستور `/contact` را بزند (یا از داخل نظرسنجی «چرا گروه رو ترک
  کردید؟»، گزینه‌ی «دلیل دیگه‌ای دارم» را انتخاب کند) و متن انتقاد یا
  پیشنهادش را بنویسد. این پیام همراه با نام و آیدی کاربر، هم برای
  `NOTIFY_CHAT_ID` و هم برای همه‌ی آیدی‌های داخل `ADMIN_IDS` فوروارد
  می‌شود. برای پاسخ، کافی‌ست شما (یا هر ادمین دیگر) روی همان پیامِ
  فوروارد‌شده در چت خصوصی‌تان با ربات **ریپلای** بزنید — متنِ ریپلای
  شما بی‌واسطه برای همان عضو ارسال می‌شود، بدون نیاز به دانستن آیدی یا
  یوزرنیمش. این نگاشت در فایل `data/contact_messages.json` نگه‌داری
  می‌شود تا حتی بعد از ریستارت شدن سرویس هم قابل استفاده بماند.
